# Salon Core (미용실 및 뷰티숍 통합 CRM 시스템)

Copyright 2026. Kimjibeom. All rights reserved.

무료 티어 빌드 기반의 미용실 및 뷰티숍 통합 CRM 시스템입니다. 예약, 고객, 매출, 직원 관리를 한 곳에서 효율적으로 처리할 수 있도록 설계되었습니다.

## 주요 기능

- **자체 예약 사이트 (New)**: 고객이 모바일/PC에서 직접 접속해 시술, 담당 디자이너, 날짜 및 시간을 선택하여 간편하게 예약할 수 있는 별도 페이지 제공
- **네이버 예약 자동 연동 (New)**: 네이버 스마트플레이스 웹훅(Webhook) 실시간 수신 및 시스템 내 직원/시술 ID 1:1 매핑을 통한 통합 DB 예약 관리
- **예약 및 대기 관리**: 일/주/월 캘린더 뷰, 당일 워크인 대기열 보드, 온/오프라인 및 네이버 예약 출처 표시
- **고객 및 차트 관리**: 전화번호 기반 고객 식별(중복 방지), 고객별 시술 히스토리북, Before/After 사진 관리, 실시간 고객 검색
- **간편 POS**: 시술/점판 항목 결제, 복합 결제 지원, 멤버십 잔액 자동 차감
- **매출 분석 대시보드**: 일/월 마감 데이터, 6개월 트렌드 차트, 객단가 분석, 신규/재방문 비율, 이탈 고객 분석
- **직원 관리**: 직원별 휴무일(Day off) 설정, 순매출 기반 인센티브 정산 및 급여 관리
- **회원권 관리**: 금액 차감형 정액권 및 횟수 차감형 회원권 통합 관리

## 처음 시작하기 (초기 설정 가이드)

시스템을 처음 설치하거나 배포한 후, 다음 순서대로 기초 데이터를 설정해야 시스템을 원활하게 사용할 수 있습니다.

1. **최초 관리자 로그인 및 설정**
   - 시스템에 로그인 후, 우측 상단 메뉴 또는 좌측 하단의 '설정' 페이지로 이동합니다.
   - `매장명`, `연락처`, `영업시간` 등 기본 정보를 입력합니다. (이 정보는 고객의 자체 예약 사이트 상단에 노출됩니다.)

2. **시술 메뉴 등록 (`/services` 또는 시술 관리 메뉴)**
   - 미용실에서 제공하는 커트, 펌, 염색 등의 시술 메뉴를 등록합니다.
   - 각 시술별 `소요 시간(분)`과 `가격`을 정확히 입력해야 자체 예약 사이트에서 예약 가능 시간을 정확히 계산할 수 있습니다.

3. **직원(디자이너) 등록 (`/staff` 또는 직원 관리 메뉴)**
   - 샵에서 근무하는 디자이너들을 등록합니다.
   - 디자이너별 `휴무일(Day off)`을 설정해 두면, 고객이 예약할 때 해당 요일은 예약이 불가능하도록 자동으로 막힙니다.

4. **네이버 예약 연동 설정 (선택 사항, `/settings/integration`)**
   - 관리자 설정의 `외부 서비스 연동 설정`에 들어갑니다.
   - 화면에 표시된 **웹훅 URL**을 복사하여 네이버 스마트플레이스 파트너센터의 예약 웹훅 설정에 등록합니다.
   - 하단의 **네이버 ID 매핑** 섹션에서, 내부 시스템에 등록한 '직원/시술'과 '네이버 시스템의 고유 ID'를 1:1로 매핑하여 추가합니다. 이 작업이 완료되어야 웹훅으로 수신된 데이터가 정상적으로 연결됩니다.

5. **자체 예약 사이트 홍보 (`/booking`)**
   - 이제 모든 준비가 완료되었습니다! 고객들에게 `https://샵도메인/booking` 주소를 안내하여 직접 예약을 받을 수 있습니다.

## 기술 스택

### Frontend
- Next.js 14 (App Router)
- React 18
- Tailwind CSS (Glassmorphism 디자인 시스템)
- Recharts (매출 분석 차트)
- TypeScript

### Backend
- Go 1.22+
- Gin-Gonic 프레임워크
- Gorilla WebSocket (실시간 예약/대기열 동기화)
- pgx (PostgreSQL 커넥션 풀링 - 무료 티어 제한 최적화)
- robfig/cron (백그라운드 알림 스케줄러)

### Database
- Supabase PostgreSQL
- Supabase Storage (미디어 업로드용)

## 아키텍처 제약 조건 및 최적화
1. **DB 커넥션 풀 제약**: Go 백엔드 DB 초기화 시 `MaxOpenConns`를 최대 20개로 제한하여 무료 티어 DB 과부하 방지
2. **미디어 스토리지 분리**: 사진/서류는 Supabase Storage에 업로드하고 DB에는 URL만 저장하여 용량 최적화
3. **콜드 스타트 대응**: Next.js API 요청 타임아웃을 60초로 설정 및 로딩 스피너 UI 구현

## 설치 및 실행 방법

### 요구사항
- Node.js 18+
- Go 1.22+
- Supabase 프로젝트 (PostgreSQL)

### 환경 변수 설정
최상단 디렉토리에 있는 `.env.example` 파일을 복사하여 `.env` 파일을 생성하고, Supabase 프로젝트 정보를 입력합니다.
```bash
cp .env.example .env
```

### 데이터베이스 마이그레이션 (필수)
초기 설치 및 업데이트 후 백엔드의 `migrations/` 디렉토리에 있는 `.sql` 파일들을 순차적으로 Supabase SQL Editor에서 실행하거나 마이그레이션 툴을 이용해 적용해야 합니다. (ex. `001_initial_schema.sql` ~ `006_unique_customer_phone.sql`) 백엔드 서버 시작 시 자동 마이그레이션도 지원됩니다.

### 백엔드 실행 (Go)
```bash
cd backend
go mod tidy
go run cmd/main.go
```
서버는 기본적으로 `http://localhost:8080`에서 실행됩니다.

### 프론트엔드 실행 (Next.js)
```bash
cd frontend
npm install
npm run dev
```
클라이언트는 `http://localhost:3000`에서 실행됩니다.

## 보안 가이드

- **JWT 비밀 키**: `JWT_SECRET_KEY` 환경 변수를 반드시 설정하세요.
- **CORS**: 프론트엔드의 도메인을 환경 변수 `FRONTEND_URL`에 명시하여 보안을 유지합니다.
- 프로덕션 배포 시에는 SSL/HTTPS 환경에서 실행되어야 합니다.
